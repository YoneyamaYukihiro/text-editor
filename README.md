# Multi-Server Log Viewer & Sora Editor

リモートサーバーのログ解析・編集を効率化する Python (PyQt6) 製ツールセット。

| アプリ | 役割 |
|---|---|
| **Multi-Server Log Viewer** ([ssh_log_viewer.py](ssh_log_viewer.py)) | 複数サーバーへ SSH/Telnet 同時接続、グリッドでログをリアルタイム表示 |
| **Sora Editor** ([text_editor.py](text_editor.py)) | 過去ログの編集・整形、Grep、SQL/Python等のシンタックスハイライト |

- バージョン: **1.0.0**
- 対応OS: Windows 10/11 (Linux/macOS も Python経由で動作)
- ライセンス: 内部利用

## 起動方法と配布

### スクリプト実行
```bash
python ssh_log_viewer.py
python text_editor.py
python text_editor.py /path/to/file.log   # 引数でファイル開けます
```

### EXE版
[dist/](dist) フォルダ内をフォルダごと配布。受け取った人は **Python不要** でダブルクリック起動。

| ファイル | サイズ |
|---|---|
| [dist/Multi-Server Log Viewer.exe](dist/Multi-Server%20Log%20Viewer.exe) | 約 42 MB |
| [dist/Sora Editor.exe](dist/Sora%20Editor.exe) | 約 36 MB |
| [dist/README.md](dist/README.md) | 取扱説明 |

> ⚠️ **両 EXE を同じフォルダに置く**こと。Multi-Server Log Viewer の「テキストエディタで開く」機能が隣の `Sora Editor.exe` を起動するため。

### EXE 再ビルド
```bash
python app_icons.py        # アイコン (ICO/PNG) 生成

python -m PyInstaller --noconfirm --noconsole --onefile \
  --name "Multi-Server Log Viewer" \
  --icon=assets/ssh_log_viewer.ico \
  --add-data "app_icons.py;." \
  ssh_log_viewer.py

python -m PyInstaller --noconfirm --noconsole --onefile \
  --name "Sora Editor" \
  --icon=assets/text_editor.ico \
  text_editor.py
```

## Multi-Server Log Viewer

### 概要
最大6サーバーへ同時接続。N×M グリッドでログを並列表示し、リアルタイム追従・SQL/レベル/タイムスタンプの構文ハイライトを行えるツール。

### サーバー管理
- **SSH/Telnet 両対応**: 接続ダイアログでプロトコル選択
- **接続プロファイル**: ホスト/ユーザー/パスワード/秘密鍵/初期DIR を保存
- **プロファイル管理ダイアログ**: ドラッグで並べ替え / ダブルクリックで名前変更 / 削除
- **6サーバー同時接続**: それぞれ識別色 (青/緑/橙/紫/赤/青緑) で視覚的に区別
- **ヘッダー表示**: プロファイル保存名を優先表示 (未保存なら `user@host`)、ツールチップにフル情報
- **クリーンアップ**: アプリ終了 / サーバー切断時に Tail プロセスと SSH/Telnet セッションを確実に閉じる

### ファイルブラウザ (左サイドバー)
- リモートDIRをツリー表示 (シンボリックリンク対応)
- パス入力 + `→` で移動 / `↻` で再読込 (ローテ後の新ファイル反映用) / `📌` で初期DIRに保存
- クイックジャンプ: `📋 ログ` / `🏠 ホーム` / `🗂 /tmp`
- **右クリックメニュー**: 「ログビューアで開く」 / 「テキストエディタで開く (DL)」
- 設定で「クイックジャンプ」「説明ラベル」の表示/非表示切替

### ログ表示グリッド
- **グリッドサイズ**: 1×1 / 1×2 / **1×3 / 1×4** / 2×1 / 2×2 / 2×4 / 3×1 / 3×2 / 3×3 / 3×4 / 4×3 / 4×4
- 各セルに **▶ Tail -F** (リアルタイム追従 + **ログローテーション自動追従**) / **■ 停止** / **↺ 再読込** / **✕ 閉じる**
- 停止ボタンは Tail 動作中のみ有効化
- セルごとのフィルタ (即時、デバウンス、正規表現対応)
- レベルフィルタ (ALL / ERR+ / WRN+ / INF+)
- ツールバーの**共通フィルタ**で全セル一括適用
- ツールバー: ▶ 全Tail / ■ 全停止 / 共通フィルタ

### 構文ハイライト (ログ用)
- **タイムスタンプ** (`2025/06/25 13:40:20.581` 等の各種形式)
- **IPアドレス**
- **ブラケット** (`[INF]`, `[main]` 等)
- **Exception/Error** クラス名
- **スタックトレース行** (`at ...`)
- **レベル背景着色** (ERROR/WARN/INFO/DEBUG/TRACE)
- **SQL構文** (ログ内に SQL 文があれば自動着色)
- **`.sql`/`.ddl`/`.dml` 等のファイル** は全文SQLモード

### 対話的ターミナル
- サーバーパネルの **`⌨ ターミナル`** ボタンから起動
- SSH (paramiko `invoke_shell`) / Telnet (新規接続) 両対応
- コマンド送信 / `Ctrl+C` 中断 / `↑↓` 履歴ナビ / ANSIエスケープ自動除去

### ワークスペース (定型ログイン)
- 「3サーバー接続 + messvr1〜4.log 開く + グリッド3×4」のような状態を**ワンクリックで復元**
- ツールバーの「ワークスペース」コンボ + `読込 / 保存 / 管理` ボタン
- **ワークスペース管理ダイアログ**: ドラッグで並べ替え / 名前変更 / 削除
- 保存内容: グリッドサイズ + 各サーバーのプロファイル名 + 開いているログパス
- パスワードをプロファイルに保存していれば**完全自動接続**

### テキストエディタ連携
- 左ツリーで右クリック → 「📝 テキストエディタで開く」
- SSH: SFTP でバイナリ取得 / Telnet: base64 経由
- **`.gz` ファイルは自動解凍**
- 一時ファイルへ保存 → `Sora Editor` を起動
- EXE版はサイドに置いた `Sora Editor.exe` を、スクリプト版は `python text_editor.py` を起動
- 一時保存先: `%TEMP%\ssh_log_viewer\<host>\<filename>`
- 起動時に 7日以上古い一時ファイルは自動削除

### 使い方 (初回セットアップ)

1. 「**＋ サーバー接続**」 → プロトコル/ホスト/ユーザー/パスワード/ログDIR を入力
2. 「**保存**」でプロファイル化 → 「**接続**」
3. 左パネルからログを **ダブルクリック** で開く
4. `▶` ボタンで Tail-F 開始

### 使い方 (ワークスペース)

1. 上記の手順で 3サーバー接続 + ログを開いた状態を作る
2. ツールバー「ワークスペース」横の **「保存」** → 名前 (例: `日次チェック`)
3. 次回以降: コンボから `日次チェック` 選択 → **「読込」**
4. 名前変更や並べ替えは **「管理」** ボタンから

## Sora Editor

### 概要
マルチタブ・FTP対応のシンタックスハイライトエディタ。Multi-Server Log Viewer から呼び出されることも想定。

### マルチタブ
- タブ ドラッグで並び替え
- タブ閉じる前に未保存編集確認
- 起動引数 `python text_editor.py <file1> <file2> ...` で開く

### シンタックスハイライト
| 拡張子 | 言語 |
|---|---|
| `.py`, `.pyw` | Python |
| `.js`, `.ts`, `.jsx`, `.tsx`, `.mjs` | JavaScript |
| `.sql`, `.ddl`, `.dml`, `.pls`, `.plsql`, `.pck`, `.prc`, `.fnc`, `.tps`, `.tpb` | SQL |
| `.log`, `messvr.log.20260414` 等 ローテーション形式 | ログ |
| その他 | 通常テキスト |

- **ログ中のSQL自動検出**: `.log` ファイル中で `SELECT`/`INSERT INTO` 等が現れる行は **SQL 構文も色付け**
- ステータスバー右下のコンボから**手動で言語切替**可能
- 文字列リテラル内の数字は数値色で上書きされない (ルール優先順位調整済み)

### 検索・置換 (Ctrl+F / Ctrl+H)
- インライン検索バー (タブ上部)
- **デバウンス + サイズ適応**: タイプ中の重い処理を回避
  - 100KB 未満: 100ms / 2MB 未満: 300ms / 10MB 未満: 700ms / 10MB 超: 1.2秒
- **ハイライト件数上限 5000件** (件数カウントは正確)
- **検索ハイライト**: 鮮黄背景 + 黒文字 + 太字 (Dark/Light どちらでも視認性良)
- **正規表現** / **大文字小文字** 切替
- **検索/置換履歴の永続化**: 最大30件保存、全タブで共有
- **📜 履歴ボタン**: 検索/置換欄の隣をクリックでドロップダウン表示
- **選択語を即検索**: `Ctrl+F3` で次の一致にジャンプ / `Ctrl+F` で選択語が自動入力

### Grep 検索
- `Ctrl+Shift+F` でフォルダ全体検索
- `*.py, *.log` のような複数globパターン対応
- 結果ツリーから該当行へジャンプ

### ブックマーク
- `F2` でマーク / `F3` / `Shift+F3` で次/前ブックマーク
- `Ctrl+B` で一覧パネル

### 編集箇所ナビゲーション (VS Code 風)
- ガター (左の縦バー) で**追加/変更/削除** 行を色分け表示
  - 緑バー = 追加 / 橙バー = 変更 / 赤バー = 削除
- **行背景も淡く着色** (より目立つ)
- `Alt+↓` / `Alt+↑` で次/前の編集箇所へジャンプ

### エラー行ナビゲーション
- `Ctrl+E` / `Ctrl+Shift+E` で ERROR/WARN/EXCEPTION を含む行へジャンプ

### 行へジャンプ
- `Ctrl+G` で行番号入力 → センタリング表示

### 全て保存 / 全て保存して終了
- `Ctrl+Alt+S`: 変更ありの全タブを一括保存 (未指定タブは「名前を付けて保存」)
- `Ctrl+Alt+Q`: 全保存 → アプリ終了 (キャンセル時は終了しない)

### 外部編集検知 (VS Code 風)
- `QFileSystemWatcher` で開いている全ファイルを監視
- **未編集タブ**: 外部で変更されたら**自動でリロード** (ステータスバー通知)
- **編集中タブ**: 「リロードしますか？」ダイアログ (デフォルトNo)
- 自分が保存した直後の自己トリガーは1秒間無視

### FTP接続
- `Ctrl+T` で FTPパネル表示
- ホスト/ユーザー/パスワード入力 → プロファイル保存可
- **サイズ事前警告**: 5MB超で確認 / 100MB超で強警告
- **ストリームDL**: メモリではなく一時ファイルへ直接書き込み (大ファイル対応)
- 一時保存先: `%TEMP%\text_editor_ftp\<host>\<filename>`
- 同名ファイル再DL時:
  - 未編集タブ → 自動でタブをリロード
  - 編集中タブ → 上書き確認ダイアログ
- 起動時に 7日以上古い一時ファイルは自動削除

### ログからSQL抽出・整形 (Ctrl+Shift+Q)
- ログ中の SQL クエリを抽出し、整形してタブで表示

## 設定の移行 (Export / Import)

別PCへ作業環境を移したいとき、設定を **JSON 1ファイル** にまとめて持ち運べます。

### Multi-Server Log Viewer
ツールバー右の **`📦 移行`** ボタン → ドロップダウンメニュー:
- **📤 エクスポート** — 接続プロファイル + ワークスペース + UI設定を JSON 保存
- **📥 インポート** — JSON 読込 (マージ / 置換 を選択)

### Sora Editor
ツールメニュー →
- **📤 設定をエクスポート...** — FTPプロファイル + UI設定 + 検索履歴 を JSON 保存
- **📥 設定をインポート...** — JSON 読込

### 動作
インポート時に3択ダイアログ:

| 選択 | 動作 |
|---|---|
| **Yes (マージ)** | 既存設定を残しつつ、同名は新しい方で上書き |
| **No (置換)** | 既存を全消去して入れ替え |
| **Cancel** | 中止 |

### PC移行手順
1. **旧PC**: 「📦 移行」→ エクスポート → 任意の場所に JSON 保存 → USB/クラウド等で運搬
2. **新PC**: アプリ起動 → 「📦 移行」→ インポート → JSON 選択 → マージ/置換選択

これでサーバープロファイル/ワークスペース/設定/履歴が **全部一括移行** できます。

> ⚠️ パスワードも JSON に平文で含まれます。USB等で運搬する際は紛失に注意。

## ファイル保存場所

すべて `C:\Users\<username>\` 直下:

| ファイル | 内容 |
|---|---|
| `.ssh_log_viewer_profiles.json` | サーバー接続プロファイル |
| `.ssh_log_viewer_settings.json` | フォントサイズ・テーマ・左パネル表示 |
| `.ssh_log_viewer_workspaces.json` | ワークスペース定義 |
| `.text_editor_ftp_profiles.json` | FTP接続プロファイル |
| `.text_editor_settings.json` | エディタ設定 (フォント/テーマ) |
| `.text_editor_history.json` | 検索/置換履歴 |

### 一時ファイル
| 場所 | 内容 | 自動削除 |
|---|---|---|
| `%TEMP%\ssh_log_viewer\<host>\` | 「テキストエディタで開く」でDLしたログ | 7日以上前は起動時に削除 |
| `%TEMP%\text_editor_ftp\<host>\` | FTPでDLしたファイル | 同上 |

> ⚠️ **パスワードは平文保存**。共有PCで使う場合は注意してください。

## トラブルシューティング

### 接続関連
| 症状 | 対処 |
|---|---|
| SSH/Telnet 接続できない | ホスト/ポート/FW確認、`ssh host` で個別確認 |
| Telnet で応答なし | サーバー認証プロンプトの表記 (`login:`/`Username:`等) を要確認 |
| パスワード入力プロンプト | プロファイルにパスワード未保存 |
| ターミナル入力が反映しない | エディタ側の入力欄にフォーカスを置く |

### Tail-F 関連
| 症状 | 対処 |
|---|---|
| Tail が止まる | `-F` なので通常自動復帰。`▶` ボタンを OFF→ON で再開 |
| ローテーション後新ファイルに飛ばない | `tail -F` が正しく動いていない場合がある (古いOSなど) |

### 表示関連
| 症状 | 対処 |
|---|---|
| 文字化け | サーバー側のロケールが UTF-8 か確認 |
| 検索が遅い | 自動でデバウンス、超巨大ファイルでは Enter のみで検索 |
| Light テーマで一部が暗いまま | 設定で再選択して OK を押す → 全UI に再適用される |

### EXE関連
| 症状 | 対処 |
|---|---|
| 初回起動が遅い | PyInstaller の解凍。2回目以降は速い |
| ウイルス対策ソフトに止められる | PyInstaller製は誤検知されやすい。除外設定追加 |
| Windows SmartScreen が出る | 署名なしEXEのため。「詳細情報」→「実行」で回避 |

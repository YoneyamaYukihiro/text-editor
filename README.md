# Multi-Server Log Viewer & Sora Editor

リモートサーバーのログ解析・編集を効率化する Python (PyQt6) 製ツールセット。

| アプリ | 役割 |
|---|---|
| **Multi-Server Log Viewer** ([ssh_log_viewer.py](ssh_log_viewer.py)) | 複数サーバーへ SSH/Telnet 同時接続、グリッドでログをリアルタイム表示 |
| **Sora Editor** ([text_editor.py](text_editor.py)) | 過去ログの編集・整形、Grep、SQL抽出/整形/実行、各種シンタックスハイライト |

- バージョン: **1.1.0**
- 対応OS: Windows 10/11 (Linux/macOS も Python経由で動作)
- ライセンス: 内部利用

## 起動方法と配布

### スクリプト実行
```bash
python ssh_log_viewer.py
python text_editor.py
python text_editor.py /path/to/file.log         # 引数でファイル開けます
python text_editor.py --search KEYWORD file.log # 起動と同時に検索
python text_editor.py --profile NAME file.log   # DB実行ダイアログで初期選択するプロファイル
```

### EXE版
[dist/](dist) フォルダ内をフォルダごと配布。受け取った人は **Python不要** でダブルクリック起動。

| ファイル | サイズ |
|---|---|
| [dist/Multi-Server Log Viewer.exe](dist/Multi-Server%20Log%20Viewer.exe) | 約 42 MB |
| [dist/Sora Editor.exe](dist/Sora%20Editor.exe) | 約 36 MB |

python -m PyInstaller --noconfirm --noconsole --onefile \
  --name "Multi-Server Log Viewer" \
  --icon=assets/ssh_log_viewer.ico \
  --add-data "app_icons.py;." \

python -m PyInstaller --noconfirm --noconsole --onefile \
  --name "Sora Editor" \
  --icon=assets/text_editor.ico \

## Multi-Server Log Viewer

### 概要
最大6サーバーへ同時接続。N×M グリッドでログを並列表示し、リアルタイム追従・SQL/レベル/タイムスタンプの構文ハイライトを行えるツール。

### サーバー管理
- **SSH/Telnet 両対応**: 接続ダイアログでプロトコル選択
- **接続プロファイル**: ホスト/ユーザー/パスワード/秘密鍵/初期DIR を保存
- **last_dir 自動記憶**: 切断時のディレクトリを次回接続時に自動復元 (手動「📌」とは別管理)
- **プロファイル管理ダイアログ**: ドラッグで並べ替え / ダブルクリックで名前変更 / 削除
- **6サーバー同時接続**: それぞれ識別色 (青/緑/橙/紫/赤/青緑) で視覚的に区別
- **ヘッダー表示**: プロファイル保存名を優先表示 (未保存なら `user@host`)、ツールチップにフル情報
- **SSH キープアライブ**: 30秒毎にパケット送信、アイドルタイムアウト切断を防止
- **自動再接続**: 通信前にトランスポート生存確認、切断検出時は1回自動再接続
- **クリーンアップ**: アプリ終了 / サーバー切断時に Tail プロセスと SSH/Telnet セッションを確実に閉じる

### ファイルブラウザ (左サイドバー)
- リモートDIRをツリー表示 (シンボリックリンク対応)
- パス入力 + `→` で移動 / `↻` で再読込 (ローテ後の新ファイル反映用) / `📌` で初期DIRに保存
- クイックジャンプ: `📋 ログ` / `🏠 ホーム` / `🗂 /tmp`
- **右クリックメニュー**: 「ログビューアで開く」 / 「テキストエディタで開く (DL)」
- 設定で「クイックジャンプ」「説明ラベル」の表示/非表示切替

### ログ表示グリッド
- **グリッドサイズ**: 1×1 / 1×2 / 1×3 / 1×4 / 2×1 / 2×2 / 2×4 / 3×1 / 3×2 / 3×3 / 3×4 / 4×3 / 4×4
- 各セルに **▶ Tail -F** (リアルタイム追従 + **ログローテーション自動追従**) / **■ 停止** / **↺ 再読込** / **✕ 閉じる**
- 停止ボタンは Tail 動作中のみ有効化
- セルごとのフィルタ (即時、デバウンス、正規表現対応)
- レベルフィルタ (ALL / ERR+ / WRN+ / INF+)
- ツールバーの**共通フィルタ**で全セル一括適用
- ツールバー: ▶ 全Tail / ■ 全停止 / 共通フィルタ

### 構文ハイライト (ログ用)
- **タイムスタンプ** (`2025/06/25 13:40:20.581` 等の各種形式)
- **IPアドレス**
- **ブラケット** (`[INF]`, `[main]` 等)
- **Exception/Error** クラス名
- **スタックトレース行** (`at ...`)
- **レベル背景着色** (ERROR/WARN/INFO/DEBUG/TRACE)
- **SQL構文** (ログ内に SQL 文があれば自動着色)
- **`.sql`/`.ddl`/`.dml` 等のファイル** は全文SQLモード

### 対話的ターミナル
- サーバーパネルの **`⌨ ターミナル`** ボタンから起動
- SSH (paramiko `invoke_shell`) / Telnet (新規接続) 両対応
- コマンド送信 / `Ctrl+C` 中断 / `↑↓` 履歴ナビ / ANSIエスケープ自動除去

### ワークスペース (定型ログイン)
- 「3サーバー接続 + messvr1〜4.log 開く + グリッド3×4」のような状態を**ワンクリックで復元**
- ツールバーの「ワークスペース」コンボ + `読込 / 保存 / 管理` ボタン
- **ワークスペース管理ダイアログ**: ドラッグで並べ替え / 名前変更 / 削除
- 保存内容: グリッドサイズ + 各サーバーのプロファイル名 + 開いているログパス
- パスワードをプロファイルに保存していれば**完全自動接続**

### アラート解析 (調査支援)
- アラートメール本文を貼り付け → **ロット/プロセス/HOST/タイムスタンプを自動抽出**
- 接続プロファイルから一致するサーバーを推定 (host/hostname/IPベース)
- 「**調査開始**」で対象サーバーへ接続 → 該当プロセスのログを **Sora Editor で直接開く**
- プロセス名のトークン分割対応: `group02_ctlsvr1` → `ctlsvr.log` 等を自動検索
- 複数候補があれば選択ダイアログ表示

### テキストエディタ連携
- 左ツリーで右クリック → 「📝 テキストエディタで開く」
- SSH: SFTP でバイナリ取得 / Telnet: base64 経由
- **`.gz` ファイルは自動解凍**
- 一時ファイルへ保存 → `Sora Editor` を起動
- EXE版はサイドに置いた `Sora Editor.exe` を、スクリプト版は `python text_editor.py` を起動
- **接続プロファイル名を引き継ぐ** (Sora Editor の DB実行ダイアログで自動選択)
- 一時保存先: `%TEMP%\ssh_log_viewer\<host>\<filename>`
- 起動時に 7日以上古い一時ファイルは自動削除

### 使い方 (初回セットアップ)

1. 「**＋ サーバー接続**」 → プロトコル/ホスト/ユーザー/パスワード/ログDIR を入力
2. 「**保存**」でプロファイル化 → 「**接続**」
3. 左パネルからログを **ダブルクリック** で開く
4. `▶` ボタンで Tail-F 開始

### 使い方 (ワークスペース)

1. 上記の手順で 3サーバー接続 + ログを開いた状態を作る
2. ツールバー「ワークスペース」横の **「保存」** → 名前 (例: `日次チェック`)
3. 次回以降: コンボから `日次チェック` 選択 → **「読込」**
4. 名前変更や並べ替えは **「管理」** ボタンから

## Sora Editor

### 概要
マルチタブ・FTP対応のシンタックスハイライトエディタ + ログ調査ツール。Multi-Server Log Viewer から呼び出されることも想定。

### クイックアクセスツールバー
メニューを開かずに主要機能を**ワンクリック起動**:

```
[📁 FTP] | [🔍 検索] [🔎 Grep] [📋 SQL抽出] | [🔖 ブックマーク]                  [⚙ 設定]
```

- **トグル式**: 同じボタンをもう一度押すと閉じる (FTP/検索/Grep/ブックマーク)
- **状態同期**: パネル/バーの実表示状態をボタンのチェック表示と連動
- ホバーで枠線が浮き上がる視覚フィードバック
- 右寄せの **⚙ 設定**

### マルチタブ
- タブ ドラッグで並び替え
- タブ閉じる前に未保存編集確認
- 起動引数 `python text_editor.py <file1> <file2> ...` で開く

### シンタックスハイライト
| 拡張子 | 言語 |
|---|---|
| `.py`, `.pyw` | Python |
| `.js`, `.ts`, `.jsx`, `.tsx`, `.mjs` | JavaScript |
| `.sql`, `.ddl`, `.dml`, `.pls`, `.plsql`, `.pck`, `.prc`, `.fnc`, `.tps`, `.tpb` | SQL |
| `.log`, `messvr.log.20260414` 等 ローテーション形式 | ログ |
| その他 | 通常テキスト |

- **ログ中のSQL自動検出**: `.log` ファイル中で `SELECT`/`INSERT INTO` 等が現れる行は **SQL 構文も色付け**
- ステータスバー右下のコンボから**手動で言語切替**可能
- 文字列リテラル内の数字は数値色で上書きされない (ルール優先順位調整済み)

### 検索 / 検索・置換
2 つを **完全分離** (Ctrl+F は検索のみ、Ctrl+H は置換欄も表示)。
- **Ctrl+F**: コンパクトな検索バー
- **Ctrl+H**: 検索 + 置換セクション (`↦` ラベルで視覚分離)
- **デバウンス + サイズ適応**: タイプ中の重い処理を回避
  - 100KB未満: 100ms / 2MB未満: 300ms / 10MB未満: 700ms / 10MB超: 1.2秒
- **ハイライト件数上限 5000件** (件数カウントは正確)
- **検索ハイライト**: 鮮黄背景 + 黒文字 + 太字 (現在行ハイライトより上に描画され、常に可視)
- **正規表現** (`.*`) / **大文字小文字** (`Aa`) のトグルボタン (押下で青く点灯)
- **検索/置換履歴の永続化**: 最大30件保存、全タブで共有 (📜 ボタン)
- **選択語を即検索**: `Ctrl+F3` で次の一致にジャンプ / `Ctrl+F` で選択語が自動入力
- **Grep との排他表示**: 一方を開くと他方は自動で閉じる

### Grep 検索
- `Ctrl+Shift+F` でフォルダ全体検索
- **コンパクトな2行UI**: 上段=フォルダ / 下段=パターン+globフィルタ+オプション+検索ボタン
- 緑強調の **▶ 検索** ボタン (主アクション)
- `Aa` / `.*` トグルで大小区別/正規表現切替
- `*.py, *.log` のような複数globパターン対応
- **マッチ箇所を黄色背景でハイライト** (QStyledItemDelegate でカスタム描画)
- **各結果行にファイル名 (basename)** を併記 (ヘッダーがスクロールアウトしてもどのファイルか分かる)
- **長い行はマッチ周辺だけ表示**: 200文字超のログ行はマッチ前後の30/200文字に切り詰め + `…`
- ホバーで完全な行をツールチップ表示
- **ジャンプは複数手段対応**: ダブルクリック / Enter / シングルクリック
- **ファイル追記によるズレを自動補正**: マッチ行テキストを記憶し、行番号がズレた場合はテキスト一致で実位置を探索
- 結果ツリーから該当行へジャンプ → エディタが自動スクロール + フォーカス取得
- **✕ ボタン / Esc キー** で閉じる

### ブックマーク
- `F2` でマーク / `F3` / `Shift+F3` で次/前ブックマーク
- `Ctrl+B` で一覧パネル (`🔖 ブックマーク一覧`)
- **✕ ボタン / Esc キー** で閉じる
- ジャンプはダブルクリック / Enter

### 編集箇所ナビゲーション (VS Code 風)
- ガター (左の縦バー) で**追加/変更/削除** 行を色分け表示
  - 緑バー = 追加 / 橙バー = 変更 / 赤バー = 削除
- **行背景も淡く着色** (より目立つ)
- `Alt+↓` / `Alt+↑` で次/前の編集箇所へジャンプ

### 現在行ハイライト (黄色統一)
- **現在行に薄い黄色オーバーレイ** (検索の黄色マッチが透過して見える)
- 左端に**黄色アクセントライン** (幅2px)
- ガター内の**行番号も黄色 + 太字**で強調
- ガター背景にも薄い黄色帯
- → カーソル位置が一目瞭然、かつ検索ヒットも隠さない

### エラー行ナビゲーション
- `Ctrl+E` / `Ctrl+Shift+E` で ERROR/WARN/EXCEPTION を含む行へジャンプ

### 行へジャンプ
- `Ctrl+G` で行番号入力 → センタリング表示
- **折り返し行があっても正確に論理行へジャンプ** (`findBlockByNumber` 使用)

### 全て保存 / 全て保存して終了
- `Ctrl+Alt+S`: 変更ありの全タブを一括保存 (未指定タブは「名前を付けて保存」)
- `Ctrl+Alt+Q`: 全保存 → アプリ終了 (キャンセル時は終了しない)

### 外部編集検知 (VS Code 風)
- `QFileSystemWatcher` で開いている全ファイルを監視
- **未編集タブ**: 外部で変更されたら**自動でリロード** (ステータスバー通知)
- **編集中タブ**: 「リロードしますか？」ダイアログ (デフォルトNo)
- 自分が保存した直後の自己トリガーは1秒間無視
- **Grep ジャンプ時にも自動リロード**: ディスクとタブ内容を比較、差分があれば更新

### FTP接続
- ツールバー **📁 FTP** ボタンまたは `Ctrl+T` で FTPパネル表示
- ホスト/ユーザー/パスワード入力 → プロファイル保存可
- **接続表示にプロファイル名優先** (明るい緑 #9ED969 太字)
- **last_dir 自動記憶**: 切断時のディレクトリを次回接続時に自動復元
- **複数選択ダウンロード**: Ctrl/Shift+クリックで複数選択 → ボタンに件数表示 → 順次DL
- **サイズ事前警告**: 5MB超で確認 / 100MB超で強警告
- **進捗ダイアログ** (大ファイル対応):
  - ダウンロード済 / 合計バイト数 / 進捗率
  - **転送速度** (例: `2.4 MB/s`)
  - **残り時間** (例: `残り 7 秒`)
  - **キャンセルボタン** で安全に中断 (一時ファイル削除)
- **ストリームDL**: メモリではなく一時ファイルへ直接書き込み (大ファイル対応)
- 一時保存先: `%TEMP%\text_editor_ftp\<host>\<filename>`
- 同名ファイル再DL時:
  - 未編集タブ → 自動でタブをリロード
  - 編集中タブ → 上書き確認ダイアログ
- 起動時に 7日以上古い一時ファイルは自動削除

## SQL抽出・整形・DB実行 (Sora Editor)

ログから SQL を取り出し、整形し、必要なら **そのまま実行して結果を取得** できる一連の機能。アラート受信 → ログ調査 → DB結果検証 のフローを1ツールで完結。

### SQL抽出・整形 (Ctrl+Shift+Q)
- ログ中の SQL 文 (Hibernate/Execute SQL/etc.) を自動抽出
- 種別バッジ表示 (`[SEL]` / `[INS]` / `[UPD]` / `[DEL]`)
- 種別ごとの色 (明るい緑/青/橙/赤) で一覧から識別容易
- 整形オプション: キーワードケース (upper/lower/cap) / インデント幅 / バインドパラメータ置換
- 整形プレビューにも SQL シンタックスハイライト
- **▶ DB実行** ボタン (緑) でそのまま DB へ投入可
- 全件1タブで開く / コピー / 個別をタブで開く

### DB実行ダイアログ (▶ DB実行)
SSH 経由でリモートサーバーのシェル上から `sqlplus` / `mysql` / `psql` 等を実行し、結果を取得。

#### 接続
- 接続プロファイルは `.ssh_log_viewer_profiles.json` を共有
- 起動元 (LogViewer) から `--profile` で渡されたプロファイルが**自動選択**
- **DB実行コマンド設定欄**は折りたたみ可能 (`▶ DB実行コマンド設定`)。通常運用ではプロファイルに保存された値で実行するため見えなくてOK

#### DBプリセット
- **Oracle (sqlplus, 通常表示)**
- **Oracle (sqlplus, CSV出力)** — 横長テーブル向け
- **Oracle (sqlplus, 縦表示)** — 列が多い場合向け
- **MySQL/MariaDB / PostgreSQL / SQLite / SQL Server (sqlcmd)**

#### 自動安全弁 (実行時に自動補正)
- **`<<EOF` → `<<'EOF'`**: bash 変数展開を防止 (`M_CONDITION$RECIPE` のような `$` 入りオブジェクト名を保護)
- **`set sqlblanklines on define off` 自動挿入**: SQL中の空行で文が切れる/`&` 変数の対話プロンプトを防止
- **`set markup csv on quote off` → `quote on`**: 値内のカンマで CSV パースが崩れるのを防止
- **SQL中の空行を自動除去**
- **改行コードを LF に正規化** (CRLF 起因の heredoc 終端不一致を防止)

#### 実行方式 (3種)
| 方式 | 用途 |
|---|---|
| **SFTP一時スクリプト + bash -l (推奨)** | `/tmp` にスクリプト書き込み → `bash --login` 実行 → 自動削除。csh/tcsh 環境でも確実 |
| **bash -l + stdin** | `bash -l` を起動し stdin 経由でコマンド送信 |
| **デフォルトシェルに直接** | リモートシェルにそのまま投げる |

#### 安全機構 (実行ブロック)
読み取り系のみ許可:
- **許可**: `SELECT / WITH / EXPLAIN / DESC / DESCRIBE / SHOW`
- **ブロック**: `INSERT / UPDATE / DELETE / MERGE / TRUNCATE / DROP / CREATE / ALTER / GRANT / REVOKE / COMMIT / ROLLBACK / SAVEPOINT / LOCK / CALL / EXEC` 等
- **`FOR UPDATE` / `FOR SHARE` 等の行ロック取得もブロック** (文字列/コメント内の偽陽性を避けるため、リテラル除去後に検出)
- 先頭コメント (`-- ...` や `/* ... */`) を除去してから先頭キーワード判定

#### 結果表示 (4 モード切替)
| モード | 内容 | 用途 |
|---|---|---|
| **📊 全件 (横・グリッド)** | 通常のテーブル | 列が少ない / 横並び比較 |
| **📋 全件 (縦・転置)** | 列名を左ヘッダー、各レコードを横列 (`#1, #2, ...`) | 列が多い + 全件一覧 |
| **🔍 1件詳細 (縦)** | 1レコードのみ縦表示 + `<< < > >>` で巡回 | 列が多い + 1件ずつ確認 |
| テキスト | 生の出力 | デバッグ |

- ヘッダーは全モードで **黄色 (#9ED969) + 太字** で統一
- **読み取り専用** (ダブルクリックでも編集不可)
- **行選択モード** (Excel貼付しやすい)
- 列幅は内容自動調整 (上限400px)

#### 出力後処理
- **「結果のみ表示」** (デフォルトON): sqlplus banner / `SQL>` プロンプト / 切断メッセージを自動除去
- **「送信コマンドも結果に表示」**: デバッグ用に実行内容を表示
- **「ドライラン」**: 実行せず生成コマンドだけ表示
- **0件警告**: 結果が空の時は ⚠ オレンジ太字でステータスバー / グリッドに通知
- **件数表示**: ステータスバー右下に `· N 件`

#### 上限件数 (UI 凍結防止)
コンボから選択: `100 / 500 / 1000 (デフォルト) / 5000 / 10000 / 無制限`
- 超過時は表示を切り詰め + ⚠ 警告表示 (SQL 自体は全件実行)
- 全件確認したい場合は SQL に `ROWNUM ≤ 1000` / `FETCH FIRST 1000 ROWS ONLY` を付与推奨

#### SQL ナビゲーション
SQL抽出ダイアログから呼ばれた場合は、DB実行ダイアログ内でも抽出済みSQLを**順次切替可**:
```
[<<] [<] [ 5 ] [/ 24] [>] [>>]
```
- 番号欄に数値入力 → Enter で任意の SQL にジャンプ
- 切替時は結果欄が自動クリア
- 親 (SQL抽出ダイアログ) の選択行も同期

#### 実行
- メインの **▶ 実行 (Ctrl+Enter)** ボタンは緑グラデーションで目立つ配色
- 状態表示: 実行中 → 完了 (exit=0): プロファイル名 · 24件

### CSV出力時の特別対応
- `set markup csv on quote on` を自動適用
- `csv.reader` でクォート付き CSV を正しくパース
- **値内に改行を含む CSV** (Oracle の複数行 COMMENTS 列等) も正しく解釈

## 設定の移行 (Export / Import)

別PCへ作業環境を移したいとき、設定を **JSON 1ファイル** にまとめて持ち運べます。

### Multi-Server Log Viewer
ツールバー右の **`📦 移行`** ボタン → ドロップダウンメニュー:
- **📤 エクスポート** — 接続プロファイル + ワークスペース + UI設定を JSON 保存
- **📥 インポート** — JSON 読込 (マージ / 置換 を選択)

### Sora Editor
ファイルメニュー →
- **📤 設定をエクスポート...** — FTPプロファイル + UI設定 + 検索履歴 を JSON 保存
- **📥 設定をインポート...** — JSON 読込

### 動作
インポート時に3択ダイアログ:

| 選択 | 動作 |
|---|---|
| **Yes (マージ)** | 既存設定を残しつつ、同名は新しい方で上書き |
| **No (置換)** | 既存を全消去して入れ替え |
| **Cancel** | 中止 |

### PC移行手順
1. **旧PC**: 「📦 移行」→ エクスポート → 任意の場所に JSON 保存 → USB/クラウド等で運搬
2. **新PC**: アプリ起動 → 「📦 移行」→ インポート → JSON 選択 → マージ/置換選択

これでサーバープロファイル/ワークスペース/設定/履歴が **全部一括移行** できます。

> ⚠️ パスワードも JSON に平文で含まれます。USB等で運搬する際は紛失に注意。

## トラブルシューティング

### 接続関連
| 症状 | 対処 |
|---|---|
| SSH/Telnet 接続できない | ホスト/ポート/FW確認、`ssh host` で個別確認 |
| Telnet で応答なし | サーバー認証プロンプトの表記 (`login:`/`Username:`等) を要確認 |
| `Socket is closed` エラー | アイドルタイムアウト切断。**再接続すれば keepalive 有効になり次回以降は防げます** |
| パスワード入力プロンプト | プロファイルにパスワード未保存 |
| ターミナル入力が反映しない | エディタ側の入力欄にフォーカスを置く |

### Tail-F 関連
| 症状 | 対処 |
|---|---|
| Tail が止まる | `-F` なので通常自動復帰。`▶` ボタンを OFF→ON で再開 |
| ローテーション後新ファイルに飛ばない | `tail -F` が正しく動いていない場合がある (古いOSなど) |

### DB実行関連
| 症状 | 対処 |
|---|---|
| `ORA-12154: TNS:could not resolve...` | テンプレートの `USER/PASS@SID` 部分を実値に書き換え |
| 結果が空 (出力ゼロ) | `feedback off pagesize 0` で 0 件が完全無出力に。CSV プリセットなら「結果のみ表示」ON で `⚠ 0 件` が出る |
| sqlplus banner が結果欄に出る | 「結果のみ表示」を ON |
| 値内のカンマで列がズレる | 自動補正で `quote on` に切り替わる。テンプレートを保存し直すと永続化 |
| 日本語条件で 0 件 | サーバーの NLS_LANG が Shift_JIS 系の場合、UTF-8 で書き込んだ SQL とミスマッチする可能性 |
| FOR UPDATE がブロックされる | 仕様。本ツールは純粋な参照のみ許可 |

### 表示関連
| 症状 | 対処 |
|---|---|
| 文字化け | サーバー側のロケールが UTF-8 か確認 |
| 検索が遅い | 自動でデバウンス、超巨大ファイルでは Enter のみで検索 |
| Light テーマで一部が暗いまま | 設定で再選択して OK を押す → 全UI に再適用される |
| Grep ジャンプが対象行に行かない | 折り返し行による論理行/視覚行の不一致は v1.1 で修正済み (`findBlockByNumber`) |

### EXE関連
| 症状 | 対処 |
|---|---|
| 初回起動が遅い | PyInstaller の解凍。2回目以降は速い |
| ウイルス対策ソフトに止められる | PyInstaller製は誤検知されやすい。除外設定追加 |
| Windows SmartScreen が出る | 署名なしEXEのため。「詳細情報」→「実行」で回避 |
